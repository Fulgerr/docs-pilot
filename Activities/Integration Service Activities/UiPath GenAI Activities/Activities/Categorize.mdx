# Categorize

# Description

Categorize content based on provided categories.

# Project compatibility

Windows | Cross-platform

# Configuration

* 
* Content - The content to categorize. This must be provided a string.
* Categories - The categories. Use the Dictionary builder widget to create a list of categories with a description and/or attributes. These are used to output an accurate category. Make sure to create clear distinctions between each category. The same category name cannot be used more than once.
* Description of content - Short description of the content being categorized. For example, product support tickets, customer reviews, etc. This field supports String type input. This field is optional. Providing a description can help with accuracy by adding more context. However, it is not always necessary and it can negatively impact the output if not carefully considered.





* Content category - The inferred category of the content.
