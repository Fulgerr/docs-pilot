# Untested Workflows

Rule ID: TA-DBP-002

Scope: Project

# Description

This rule checks for workflows that do not have at least one assigned test case.

# Recommendation

Enable this rule to discover potential issues early on in the testing stage.

# Configure Rule

1. In the Design ribbon, navigate to Analyze File > Workflow Analyzer Settings.
2. Go to Workflow Analyzer to view the rules.
3. Select Untested Workflows and configure the default action.
4. Click OK to save changes.

# Reset to Default Value

To reset the rule to the default configuration, right-click it and select Reset to default.
