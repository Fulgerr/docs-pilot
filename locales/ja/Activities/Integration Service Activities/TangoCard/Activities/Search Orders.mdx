# Search Orders

# Description

Search orders that fulfill specific criteria.

# Project compatibility

Windows | Cross-platform

# Configuration

* 
* Account - The account to be queried. You can select an account from the available drop-down list. This field supports String type input.
* Customer - The customer to be queried. You can select a customer from the available drop-down list. This field supports String type input.
* Minimum amount - Specify the minimum face value of the reward to be queried. This field supports Double type input.
* Maximum amount - Specify the maximum face value of the reward to be queried. This field supports Double type input.
* From - Specify the starting date or date time to be queried. Use the calendar widget to set a date. This field supports DateTimeOffset type input.
* To - Specify the ending date or date time to be queried. Use the calendar widget to set a date. This field supports DateTimeOffset type input.
* Currency code - Specify the currency code of the reward to be queried. This field supports String type input.
* Order status - Specify the status to be queried. Select an option from the available drop-down list: Complete, Pending, Failed, Canceled, Partial.
* Reward name - Specify the reward name of the reward to be queried. This field supports DateTimeOffset type input.







* Sender's first name - Use this field to set the sender's name and email address if different than the default. This field supports String type input.
* Sender's last name - Use this field to set the sender's name and email address if different than the default. This field supports String type input.
* Recipient's first name - The given name of the individual receiving the reward. This field supports String type input.
* Recipient's last name - The surname of the individual receiving the gift card. This field supports String type input.



* Orders - Automatically generated output variable.
